## 关键问题
1. 如何只通过语音描述创作复杂图形, 同时保持对象可编辑和可撤销。
2. 生图模型应该放在核心绘图路径, 还是作为素材和背景增强。

## 已做决策
| 决策 | 理由 |
|------|------|
| 优先升级矢量对象模型和复杂指令 DSL | 保持纯语音绘图的可编辑性、低延迟和可撤销能力 |
| 生图模型作为素材和背景增强层 | 文生图适合丰富视觉, 但不适合承担精确对象编辑主路径 |
| 短期不立即迁移画布底座 | 当前先稳定领域对象模型, 再评估 tldraw 或 Excalidraw 迁移收益 |
| 阶段 A 先做兼容式对象元数据扩展 | 比直接重写画布底座风险低, 并能立即支撑复杂语音指令 |
| 阶段 B 同时做路径对象和评测集 | 路径对象提升复杂形状表达, 评测集防止复杂指令能力退化 |
| 阶段 C 先追问再执行复杂场景 | 多主体场景如果无法安全拆解, 不应被规则解析器误画成半成品 |
